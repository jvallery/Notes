## Summary

The team reviewed the updated VAST 5.5 release plan with feature freeze targeted for the week of 2025-11-10, beta targeted for January 2026, and restricted release targeted for March 2026, with debate on the typical restricted-to-GA gap (generally 5-6 weeks). Discussion emphasized technical debt and resilience work (NDU latency, install fixes, support bundles, metrics scaling, testability), clarified several 5.5 scope decisions (Query Engine DDL in, group-by out; multiple items deferred to 5.6), and covered hardware change management and supply risks (NICs and drives) requiring risk buys and alternate sourcing.

## Decisions


- VAST Data 5.5 includes Azure GA as part of the 5.5 release scope.

- Cloud decommission functionality is delivered in VAST Data 5.5 for cloud flows.

- Query Engine DDL is included in VAST Data 5.5.

- Query Engine group-by aggregations are not included in VAST Data 5.5.

- SMB scale is deferred from VAST Data 5.5 to VAST Data 5.6.

- SPDK target work is moved from VAST Data 5.5 to VAST Data 5.6.

- Database projections are moved from VAST Data 5.5 to VAST Data 5.6.

- NFSv4 delegations view-level is added in VAST Data 5.5.

- Physical quota is added in VAST Data 5.5.

- SMB server-side copy (SMBX copy) is added in VAST Data 5.5.

- Broadcom 4.2 is not currently in VAST Data 5.5 scope and requires a separate justification to evaluate.




## Key Information


- VAST Data release 5.5 plan targets feature freeze during the week of 2025-11-10, beta in January 2026, and restricted release in March 2026.

- The team stated the typical restricted-release to GA gap for VAST minor releases is about 5-6 weeks, and debated whether 5.5 should follow that cadence.

- VAST Data 5.5 prioritizes technical debt and resilience improvements including NDU latency improvements, installation fixes, and expanded support bundle capabilities requested by Support and customers.

- VAST Data 5.5 includes metrics scaling work and improved e-box qualification and testability work in AVMS/VMS areas.

- Asaf Levy performed field research that drove several platform and control fixes/features included in VAST Data 5.5.

- The 'old snapshots' feature discussed for VAST Data 5.5 was described as a secondary access point to snapshots taken independently, not merely hiding snapshots from view.

- Software-defined e-box enablement goal is to add new e-box hardware without requiring VMS or platform code changes, with a further move toward config-file based enablement planned for VAST Data 5.6.

- Non-standard e-box hardware qualification time varies based on deviation size; examples referenced included CloudStrike and Tesla in VAST Data 5.4, with larger changes (e.g., TCP instead of RDMA) taking longer.

- Cloud scope for VAST Data 5.5 includes Azure GA, while AWS and GCP have minor releases without decommission; decommission is delivered in 5.5 for cloud flows.

- Query Engine scope for VAST Data 5.5 includes DDL support, while group-by aggregations are not included in 5.5 and performance is the focus.

- SMB scale, SPDK target work, and database projections were explicitly moved out of VAST Data 5.5 and deferred to VAST Data 5.6.

- MD Migrate was flagged as risky for a minor release (VAST Data 5.5) and may require a limited customer rollout with disable and rollback safety.

- NFSv4 event triggers for HTX were discussed as targeting a December 2025 beta, with concern about readiness for an early-December dark-site production install.

- Hardware change management for VAST Data 5.5 includes firmware updates, BOM finalization, sample quantity planning, and risk buys to secure December 2025 QA quantities; supply risk exists for NICs (NVIDIA CX7) and drives (SCM and 122 U.2).
