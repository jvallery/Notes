# Cisco RFE 0526: Regex-based allow/deny patterns for replication policies

**Date**: 2025-12-17
**Account**: [[]]

## Summary

Brian Evans notified the PM team that Cisco submitted and received approval for RFE 0526 (Existing Feature Enhancement). The request is to add regex or pattern-based allow/deny filtering of files/objects by name within replication policies to support Cisco's many global replication relationships.

## Key Information


- Brian Evans approved RFE 0526 (Existing Feature Enhancement) submitted to the VAST Data PM team for Cisco.

- Jeff Mohler is the Sales Engineer associated with Cisco RFE 0526.

- Cisco operates many global sites with data replication across hundreds of replication relationships.

- Cisco requests the ability to use regex or pattern matching to allow or deny files/objects by name in replication policies, for example allow only *.LIB and deny *.mp3.

- Cisco's stated value proposition for RFE 0526 is multi-geo liability protection by preventing replication of copyrighted content using a small set of 'never replicate' patterns.
